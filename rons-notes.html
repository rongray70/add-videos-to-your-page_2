OOP is a design wherein you define classes and methods rather than specific elements. For example, you might create a 'dog' class which defines various attributes (hair length, temperment) and methods to interact with an element (walk, feed, play). You can now use this class to create as many instances/objects of a dog as you need. Without OOP you would be defining the doggy attributes/methods each time you created a new 'dog', but with OOP you need only do that once in a class. This makes alterations much quicker and simpler, as alterations or additions can be made in one class which will then affect all of the instances using that class. For example, you could add a 'size' attribute, which would then become available to all instances of all dogs. Without OOP and classes you would have to make changes to the definitions of each instance. It's very similar to CSS... rather than defining the particular attributes you want for each and every paragraph in your html document you can simply create a paragraph style class which defines how all instances of a paragraph should appear (normal weight, 10px, etc). Should you decide that all paragraphs need to be sans serif, then only one change is needed (to the paragraph class) and all paragraphs will subsequently have the change.

Inheritance occurs when subclasses inherit the features of a parent class. This can be done with class variables as well as class methods. The simplist example I know of relates back to cascading style sheets. A body element might have a definition for a particular font (Times New Roman). All of the sub-elements of body (headings, paragraphs, etc) will inherit this font selection. This saves the necessity of having to explicitly define the font for each sub-element. It (again) also allows for quick changes if the developer decides to use Arial instead... one change is made to the body font-family and all sub-elements subsequently use that font. As with python class variables and methods, specific selections can still be overwritten should the developer choose to do so.

Network: group of entities that can communicate, even though they are not all directly connected

Latency: time that it takes for a message to travel from source to destination

Bandwidth: amount of information that can be transmitted per unit time; rate of data which can be transmitted (ie, bps)

Bit: smallest unit of information

Protocol: set of agreed-upon rules which dictate how two entities talk to each other. An example is http.

URL: uniform resource locator

query parameter: parameters which can be passed in the URL (?x=1&y=2...)

fragment: indentified in query parameter using hash (#) and occurs at end of list; references a particular part of the page; not sent to server, used by browser

http has a method, path, and version: for example
http://www.example.com/foo
GET /foo HTTP/1.1 is sent to www.example.com on port 80

An http request generally contains multiple headers:
host: indicates the host server/name/address
user-agent: indicates the app used to make the request (chrome, ie, ffox)

http responses are sent back from the server; example response:
HTTP/1.1 200 OK

Responses can also contain multiple headers:
Date: indicates server's timestamp
Server: name/version number of server app (ie, Apache/2.2.3)
Content-Type: type of data returning (ie, text/html)
Content-Length: length of content

Purpose of a 'server' is to respond to requests

Types of responses:
static: existing file or image
dynamic: response created based on request, generally created by web application
